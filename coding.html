Bloom Filter

Implemented insert, rebuild, "hack", checking/search, and hash functions for a Bloom Filter. Insertion was preceded by a "hack" function, which was used to simulate a system that keeps track of unsuccessful login attempts. After some pre-defined number of calls to hack() for a given username, the username is then passed to the insert function. Insert() calls each hash function to determine which indices of the bit array should be set to 1. Upon successful insertion, insert() checks whether the false-positive probability is within the pre-defined limits; if it is not, the rebuild function is called, finding a new m value (length of bit array), making a new bit array of length new m, and reinserting all previously-inserted usernames. Lastly, the checking/search function calls the hash functions in a similar manner to insert(), but instead verifying that the respective bits are already set to 1. As with all Bloom Filters, there is an unavoidable probability that check() will return a false-positive. Written in Python. Completed as part of the curriculum for CMSC420 (Advanced Data Structures) during the Spring 2023 Semester. Per academic integrity guidelines, I cannot post the code publicly.

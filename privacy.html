Privacy Notice

The code I have written for this site ( github.com/psmit703/personal-website/tree/main ) makes limited use of non-identifiable information:

  • The Home Page uses JavaScript to determine the height and width (in pixels) of the user's browser window in order to determine whether or not to display an image. If the aspect ratio is too small, then no image is displayed. This is done to create, in my opinion, a better aesthetic on mobile devices. All of the processing is done by the user's browser and nothing is stored or sent back to me (Pete Smith) in any way.
  • The Changelog Page uses JavaScript to retrieve recent commits to the site's GitHub repository. It does this by interacting with GitHub's REST API through Octokit.js, which itself is maintained by GitHub and is GitHub's official way of interacting with their REST API. The code that I have written processes solely on the user's browser and nothing is stored or sent back to me (Pete Smith) in any way. However, as this interacts with GitHub's REST API, I cannot guarantee that anything related to data-privacy is true regarding GitHub. Please see GitHub's privacy policy ( docs.github.com/en/site-policy/privacy-policies/github-privacy-statement ) for more information.
  • The Error 404 Page uses JavaScript to determine the URL that a user has inputted. As this page is an error page, the URL is used in a message displayed to the user. All of the processing is done by the user's browser and nothing is stored or sent back to me (Pete Smith) in any way.
  • Every page uses JavaScript for certain navigation bar features. There are three scripts that control this behavior:
    • The first, which controls whether the navigation bar is visible, uses JavaScript to determine whether or not a user is scrolling and where the mouse is relative to the browser's viewport. These are used to determine whether or not the navigation bar should be displayed. All of the processing is done by the user's browser and nothing is stored or sent back to me (Pete Smith) in any way.
    • The second script controls the navigation bar menu animation when the site is viewed in mobile format. This script uses JavaScript to determine whether or not a user has clicked (or pressed for touchscreens) the navigation bar menu icon. This is used to apply a CSS rotation animation to the menu icon. This script also controls whether or not a click/press will close the navbar menu in mobile format. If the navbar menu is open and the site is being viewed in mobile format, then clicking/pressing outside of the navbar menu will close the navbar menu. All of the processing is done by the user's browser and nothing is stored or sent back to me (Pete Smith) in any way.
    • The third script controls the text alignment of navbar items. The default state is center-alignment, which appears in desktop mode. This allows for the bold effect of the active page to expand both to the left and the right, rather than just the right. However, in mobile format, center-alignment would make it centered relative to the screen. As such, this script uses JavaScript to change the navbar items to left-aligned when the site is viewed in mobile format. All of the processing is done by the user's browser and nothing is stored or sent back to me (Pete Smith) in any way.
  • The entire site has a toggleable dark mode. Dark mode is automatically enabled/disabled depending on a user's device settings. If a user manually toggles dark mode on or off, this preference is stored using a cookie. This cookie is set to expire automatically after 30 days. Additionally, there is a clickable option at the bottom of each page that allows a user to manually delete this cookie. The cookie is stored on the user's device, all of the processing is done by the user's browser, and nothing is sent back to me (Pete Smith) in any way.
Other than what has already been stated on this page, the code I have written for this site does not process, store, or otherwise use any user data in any way and does not otherwise use cookies. However, this site makes use of various third party software that may do so. The following are short summaries of relevant privacy statements and links to their respective privacy policies, where applicable:

  • Bootstrap
    • Bootstrap does not appear to have a privacy policy, and, as far as I am aware, the service itself does not collect or process user data
    • However, Bootstrap's components are imported from two different CDNs when the site is loaded:
      • jsDelivr
        • jsDelivr's full privacy policy is available at www.jsdelivr.com/terms/privacy-policy-jsdelivr-net
        • As of July 10, 2023, jsDeliver, in short, does not collect personal data or use cookies, but may collect various usage statistics such as IP addresses and browser information
      • jQuery
        • jQuery's full privacy policy is available at openjsf.org/wp-content/uploads/sites/84/2021/04/OpenJS-Foundation-Privacy-Policy-2019-11-15.pdf
        • As of November 21, 2019, jQuery, in short, may collect usage statistics such as IP addresses and browser information through the use of cookies, web beacons, and "other technologies," along with collecting de-identified information for analysis and improvement purposes
  • GitHub Pages | GitHub REST API | Octokit.js
    • GitHub's full privacy policy is available at docs.github.com/en/site-policy/privacy-policies/github-privacy-statement
    • As of July 10, 2023, GitHub offers the following summary:
      • When a GitHub Pages site is visited, the visitor's IP address is logged and stored for security purposes, regardless of whether the visitor has signed into GitHub or not. For more information about GitHub's security practices, see GitHub Privacy Statement.
    • Note: Octokit.js is maintained by GitHub and is GitHub's official way of interacting with their REST API
  • Plausible Analytics
    • Plausible's full privacy policy is available at plausible.io/privacy
    • Plausible's full data policy is available at plausible.io/data-policy
    • Plausible is open-source and the code is available at github.com/plausible/analytics
    • Plausible is a privacy-friendly alternative to Google Analytics
      • Plausible is compliant with GDPR, CCPA, and PECR
      • Plausible collects, processes, and stores limited, non-identifiable information that allows for site administrators to track data such as view counts and city-level location data
      • Plausible does not use cookies
  • Clicky Website Analytics
    • Clicky's full privacy policy is available at clicky.com/terms/privacy
    • Clicky is a privacy-friendly alternative to Google Analytics
      • Clicky is compliant with GDPR
      • Clicky collects, processes, and stores limited, non-identifiable information that allows for site administrators to track various statistics such as the number of unique visitors and total page views
      • You can opt out of Clicky's tracking for all sites that use the service by visiting clicky.com/optout and following the instructions at the bottom of the page
        • This uses a cookie that is valid for 1 year, which is necessary in order for site's using Clicky to recognize your preference
        • This preference can be overridden by sites that use Clicky for security and fraud prevention purposes, however this site does not use Clicky for any such purposes, therefore, if you opt out of tracking, this site will honor your preference
      • Clicky may use cookies if privacy settings are disabled or if their is a user consent system in place
        • Privacy settings are enabled for all users viewing this site and there is no consent system currently in place on this site, therefore cookies are not used by Clicky for this site
